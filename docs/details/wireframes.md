# 画面ワイヤーフレーム（詳細）

[← 要件定義書に戻る](../requirements.md)

各画面の要素配置（ブロック構造）を Mermaid で示す。実際のレイアウト（余白・サイズ・色）は別途デザインで決定する。

## S-01 ボード画面

```mermaid
flowchart TB
    subgraph S01[S-01 ボード画面]
        H01[ヘッダー: ロゴ / ユーザー名 / ログアウトボタン]
        subgraph Board[カンバンボード（左から右へ進行）]
            direction LR
            C1[やること列<br/>カード一覧<br/>+ カード追加ボタン] --> C2[進行中列<br/>カード一覧] --> C3[完了列<br/>カード一覧]
        end
        F01[フッター: + リスト追加ボタン / 完了タスク一覧へ ボタン]
        H01 --> Board --> F01
    end
```

## S-02 カード詳細画面（モーダル）

ボード画面のカードをクリックすると、ボード画面の上に重ねてモーダル表示される。背景はオーバーレイで暗くする想定。

```mermaid
flowchart TB
    subgraph S02[S-02 カード詳細画面 ／ モーダル表示]
        H02[ヘッダー: × 閉じるボタン]
        T02[タイトル入力欄]
        D02[説明文入力欄（複数行）]
        DL02[期限入力欄（日付ピッカー）]
        P02[優先度選択（高 / 中 / 低）]
        F02[フッター: 保存ボタン / 削除ボタン / キャンセルボタン]
        H02 --> T02 --> D02 --> DL02 --> P02 --> F02
    end
```

## S-05 完了タスク一覧画面

画面を開いた時点では、完了したタスクを **完了日が新しい順（最新順）** で一覧表示する。
画面上部に1行分の履歴検索エリアを配置し、キーワードを入力すると一覧が検索結果に絞り込まれる。
画面の大部分は完了タスク一覧の表示領域として使い、戻るボタンはフッターに配置する。

各タスクは **タイトル・完了日・説明文（先頭1〜2行のみ省略表示）** を一覧に表示する。
直感的に完了タスクの内容が把握できることを目的とし、行をクリックすると S-02 カード詳細画面（モーダル）が開いて説明文の全文を確認できる。

```mermaid
flowchart TB
    subgraph S05[S-05 完了タスク一覧画面]
        H05[ヘッダー: 画面タイトル「完了タスク一覧」]
        subgraph Search[履歴検索エリア（1行・コンパクト）]
            direction LR
            T05[タイトル検索ボックス]
            D05[説明文検索ボックス]
            B05[検索ボタン]
        end
        R05[完了タスク一覧（メイン領域・大きく表示）<br/>完了日が新しい順に表示<br/>各行: タイトル / 完了日 / 説明文の先頭1〜2行（省略表示）<br/>行クリックで S-02 カード詳細モーダルを開く]
        F05[フッター: 戻るボタン]
        H05 --> Search --> R05 --> F05
    end
```
