# 業務フロー図（詳細）

[← 要件定義書に戻る](../requirements.md)

対象範囲：フェーズ4までの個人利用シーン（ログイン〜タスク完了〜履歴振り返り）。

## メイン業務サイクル

タスクの発生から完了までの基本的な業務の流れ。

```mermaid
flowchart TD
    subgraph U[ユーザー]
        U1[タスクの発生に気づく]
        U2[アプリにログイン]
        U3[新規カードを追加<br/>タイトル・期限・優先度を入力]
        U4[着手時にカードを<br/>「進行中」列へ移動]
        U5[完了時にカードを<br/>「完了」列へ移動]
    end

    subgraph S[システム]
        S1[認証処理]
        S2[ボード画面を表示]
        S3[カードを「やること」列に<br/>保存・表示]
        S4[カードの状態を更新・保存]
        S5[完了日を自動記録し<br/>アーカイブとして保存]
    end

    Start([業務開始]) --> U1
    U1 --> U2 --> S1 --> S2
    S2 --> U3 --> S3
    S3 --> U4 --> S4
    S4 --> U5 --> S5
    S5 --> End([業務終了])
```

## 任意の付随作業（履歴確認）

メイン業務サイクルの終了後、必要に応じて実施する任意の作業。
過去に完了したタスクの事実を後から思い出す／確認するために行う。

```mermaid
flowchart TD
    subgraph U2g[ユーザー]
        UA1[過去の完了タスクを<br/>確認したくなる]
        UA2[履歴検索画面を開き<br/>キーワードを入力]
        UA3[結果一覧から<br/>該当タスクを確認]
    end

    subgraph S2g[システム]
        SA1[履歴検索画面を表示]
        SA2[条件に合致する<br/>完了履歴を一覧表示]
    end

    StartA([確認したいときに開始]) --> UA1
    UA1 --> UA2 --> SA1
    SA1 --> SA2 --> UA3
    UA3 --> EndA([確認終了])
```

## 補足

- フェーズ1〜2では「ログイン」を伴わず、ブラウザ上で直接ボード画面に到達する
- フェーズ5（将来）では、上記フローに加えて「グループメンバーへ共有」「他メンバーが編集」というステップが追加される
