# タスク管理アプリ 要件定義書

## 1. プロジェクト概要

### 1.1 アプリ名
TaskManagement（仮称）

### 1.2 目的
個人のタスクを視覚的に管理し、進捗状況を一目で把握できるようにする。
将来的にはグループでのタスク共有を可能にし、チームでの作業管理にも対応する。

### 1.3 背景
- 日々のタスクを整理し、やるべきことを見える化したい
- Trello風のカンバン方式で直感的に操作できるアプリを自作することで、Web開発の基礎を学ぶ

### 1.4 対象範囲（スコープ）

本プロジェクトで扱う範囲（スコープ内）と、本要件定義書では扱わない範囲（スコープ外）を対比して明示する。後続の解釈の食い違いやトラブルを防ぐため、特にスコープ外を明確に記載する。

#### 1.4.1 機能面

| スコープ内 | スコープ外 |
|----------|-----------|
| カードの追加／表示／移動／編集／削除／アーカイブ | 添付ファイル機能 |

| 期限・優先度の設定、期限警告色 | カレンダー表示・ガントチャート |

## 4. 業務要件

### 4.1 業務概要
本アプリのユーザーは、日常的に発生するタスクを「登録 → 着手 → 完了」という基本サイクルで管理する。アプリは各ステップで必要な情報の保存・表示を担い、ユーザーが進捗を一目で把握できるよう支援する。また、業務終了後に必要に応じて過去の完了タスクを履歴確認できる。

### 4.2 業務フロー図

メイン業務サイクル（登録 → 着手 → 完了 → 業務終了）と、業務終了後の任意作業（履歴確認）の2枚を Mermaid 図で示す。

詳細は [`details/business-flow.md`](details/business-flow.md) を参照。

### 4.3 ユースケース図

アクターとユースケース（業務単位）の関係を Mermaid で示す。フェーズ1〜4（ユーザー1名）とフェーズ5（ユーザー／グループメンバー／グループリーダーの3アクター）の2パターン。

詳細は [`details/use-cases.md`](details/use-cases.md) を参照。

### 4.4 ユースケース記述

ユースケース図に登場する全10件（UC-01〜UC-10）を、標準形式（アクター／概要／事前条件／事後条件／基本フロー／代替フロー）で記述する。フェーズ1〜4（UC-01〜UC-05）とフェーズ5（UC-06〜UC-10）を含む。

詳細は [`details/use-cases.md`](details/use-cases.md) を参照。

### 4.5 補足
- フェーズ1〜2では「ログイン」を伴わず、ブラウザ上で直接ボード画面に到達する
- フェーズ5（将来）では、上記フローに加えて「グループメンバーへ共有」「他メンバーが編集」というステップが追加される

## 5. 機能要件

### 5.1 フェーズ1: MVP（最小限の機能）
| ID | 機能名 | 概要 |
|----|--------|------|

| F-02 | カード表示 | 追加したカードが画面上に表示される |
| F-03 | カード移動 | 「やること」「進行中」「完了」の列間を移動できる |
| F-04 | カード削除 | 不要になったカードを削除できる |

### 5.2 フェーズ2: 基本機能の拡張
| ID | 機能名 | 概要 |
|----|--------|------|
| F-05 | カードタイトル編集 | カードのタイトルを後から変更できる |
| F-06 | カード説明文 | カードに詳細な説明文を追加できる |
| F-07 | 期限設定 | カードに期限（日付）を設定できる |
| F-08 | データ保存 | ブラウザを閉じてもデータが残る |
| F-09 | ドラッグ&ドロップ | カードをマウス操作で移動できる |
| F-10 | リスト追加・削除 | 列（リスト）を任意に追加・削除できる |
| F-11 | 完了タスクのアーカイブ | 「完了」列に移動した時点で完了日を記録し、削除しても履歴データとして保存される |

### 5.3 フェーズ3: ユーザー機能
| ID | 機能名 | 概要 |
|----|--------|------|
| F-12 | ユーザー登録 | ニックネーム・メールアドレス・パスワードで新規登録できる |

| ID | 機能名 | 概要 |

## 6. 非機能要件

IPA「非機能要求グレード」の6大項目（可用性／性能・拡張性／運用・保守性／移行性／セキュリティ／システム環境）に準拠して整理する。

| 項目 | 内容 |

### 6.2 性能・拡張性

| 項目 | 内容 |

## 8. 使用技術・システム構成

### 8.1 フェーズ1〜2

フェーズ3以降と同じフロントエンドスタックを採用し、バックエンドだけ持たない構成とする。フェーズ3への移行時にコードの書き直しが不要になる。

- **言語**: TypeScript 6.0
- **ライブラリ**: React 19
- **ビルドツール**: Vite 8
- **ルーティング**: React Router
- **データ保存**: localStorage（フェーズ3でバックエンドAPI 呼び出しに差し替え）
- **Lint / フォーマット**: ESLint + Prettier
- **テスト**: Vitest + React Testing Library（任意）
- **動作確認**: 開発時は `npm run dev`（Vite 開発サーバ）。配信が必要なら `npm run build` で静的ファイル化し、GitHub Pages 等に配置可能

※ 現在 `prototype/` 配下に置かれている HTML / CSS / JavaScript の実装は、要件確認用の使い捨てプロトタイプである。本実装のフェーズ1は新たに React で構築し、プロトで得た知見（画面構成・操作感・データ構造）を反映する。

- **言語**: TypeScript 6.0

- **ビルドツール**: Vite 8
- **ルーティング**: React Router

- **テスト**: JUnit 5 + Mockito + Spring Boot Test
- **API スタイル**: REST API（JSON over HTTP、HTTPメソッドでCRUD操作を表現）

- **バージョン管理**: Git / GitHub

## 9. データ仕様

### 9.1 データ項目

各エンティティの業務的な属性を記載する。物理的な型・制約の詳細はER図（9.2）を参照。

#### 9.1.1 タスクカード（CARD）

| 項目 | 型 | 必須 | 備考 |
|------|----|----|------|
| ID | 文字列 | ○ | 一意の識別子 |
| タイトル | 文字列 | ○ | カードの見出し |
| 説明文 | 文字列 | × | 詳細メモ |
| 期限 | 日付 | × | 締め切り日 |
| 優先度 | 列挙（高・中・低） | × | 並び替え・表示用 |
| 状態 | 列挙（やること・進行中・完了） | ○ | 所属する列 |
| 完了日 | 日付 | × | 完了列に移動した日付を記録 |
| アーカイブ済み | 真偽値 | ○ | 削除済みかどうか（履歴として残すため削除フラグ方式） |
| 並び順 | 数値 | ○ | リスト内での表示順 |

#### 9.1.2 その他のエンティティ

リスト（LIST）、ユーザー（USER）、ボード（BOARD：フェーズ5）、グループ（GROUP：フェーズ5）、グループメンバー（GROUP_MEMBER：フェーズ5）のデータ項目は [`details/data-model.md`](details/data-model.md) にまとめている。
